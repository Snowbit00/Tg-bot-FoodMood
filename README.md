# Автоматизированный Telegram-канал «Рецепты»

Полностью пассивный канал: **DeepSeek** пишет рецепты, **YandexART** рисует фото блюда,
бот публикует пост в Telegram по расписанию. Один запуск скрипта = один пост.

- Текст — прямой API **DeepSeek** (`DEEPSEEK_API_KEY`).
- Картинки — **YandexART** в Yandex AI Studio (`YANDEX_API_KEY` + `YANDEX_FOLDER_ID`).
- Провайдер текста настраивается в `config.yaml` (можно перевести и на Yandex AI Studio).

## Как это работает

```
Планировщик Windows (09:00 / 14:00 / 19:00)
   └─ python generate_and_post.py
        ├─ выбирает рубрику дня (config.yaml → rubricator)
        ├─ DeepSeek (api.deepseek.com) → рецепт в JSON
        ├─ YandexART → фото блюда по image_prompt
        ├─ форматирует под лимиты Telegram (caption ≤ 1024)
        ├─ Bot API: sendPhoto (+ sendMessage при длинном рецепте)
        └─ дописывает блюдо в state/posted.json (антиповтор)
```

## Установка

1. **Python 3.10+** (`python --version`).

2. Зависимости:
   ```powershell
   cd "C:\ИИ тг канал"
   python -m pip install -r requirements.txt
   ```

3. Секреты: скопируйте `.env.example` → `.env` и заполните:
   ```powershell
   copy .env.example .env
   ```
   - `DEEPSEEK_API_KEY` — ваш ключ DeepSeek (текст).
   - `YANDEX_API_KEY`, `YANDEX_FOLDER_ID` — картинки (YandexART).
   - `TELEGRAM_BOT_TOKEN` — бот от **@BotFather**, добавлен **администратором** канала.
   - `TELEGRAM_CHANNEL_ID` — `@username` канала или числовой `-100…`.

## Настройка YandexART (только для картинок)

1. Привязать платёжный аккаунт в Yandex Cloud (без него модели не работают; новым дают грант).
2. Сервисный аккаунт с ролью **`ai.imageGenerationModels.user`** (для текста через Yandex
   добавьте ещё `ai.languageModels.user`).
3. «Создать API-ключ» для этого сервисного аккаунта → `YANDEX_API_KEY` (показывается один раз).
4. Идентификатор каталога `default` → `YANDEX_FOLDER_ID`.

## Проверка перед автономным режимом

```powershell
# 1. Только текст — проверить ключ DeepSeek и модель
python generate_and_post.py --dry-run --skip-image

# 2. Полное превью с картинкой (сохранится в out\preview.jpg), без публикации
python generate_and_post.py --dry-run

# 3. Боевая публикация в ТЕСТОВЫЙ канал, затем 5–6 раз подряд —
#    убедиться, что блюда не повторяются (работает антиповтор)
python generate_and_post.py
```

Логи — в `logs\run.log`.

## Ручной запуск (по команде или двойным кликом)

Канал работает в двух режимах одновременно — по расписанию и вручную:

- **Командой** (из папки проекта): `python generate_and_post.py` — сразу публикует один пост.
- **Двойным кликом** (без терминала):
  - `post_now.bat` — опубликовать пост прямо сейчас;
  - `preview.bat` — превью без публикации (рецепт + картинка в `out\preview.jpg`).

Ручной запуск и расписание используют один и тот же скрипт и не мешают друг другу.

## Управление с телефона (команды боту)

Можно публиковать удалённо — написав команду боту в личном чате.

**Настройка (один раз):**
1. Узнать свой Telegram ID: напишите боту `/start` — он ответит «Ваш ID: …»
   (или спросите у `@userinfobot`).
2. Впишите ID в `.env`: `TELEGRAM_ADMIN_IDS=ваш_id` (несколько — через запятую).

**Запуск бота (всегда онлайн, с авто-перезапуском):**
```
py launcher.py start
```
или двойной клик **`launcher.bat`**. Супервизор держит бота запущенным и перезапускает
его при падении.
- Остановить: `py launcher.py stop`
- Статус: `py launcher.py status`

**Чтобы стартовал при входе в Windows:** `Win+R` → `shell:startup` → положите туда ярлык
на `launcher.bat`.

**Команды (в личке с ботом, только для админов):** доступны через **кнопки** под полем
ввода и через **меню «Menu»** / автодополнение `/`:
- `/new` — **создать рецепт с превью и одобрением** (см. ниже). Можно с рубрикой: `/new супы`.
- `/post [рубрика]` — сразу сгенерировать и опубликовать рецепт (без превью).
- `/poll` — опубликовать опрос-вовлечение.
- `/pollresults` — текущие голоса опроса.
- `/preview` — быстрый текст-превью без публикации.
- `/help` — список команд и ваш ID.

**Режим `/new` (превью → одобрение):**
1. Бот генерирует черновик рецепта (текст, без картинки) и показывает его с кнопками
   **👍 Нравится · 👎 Не нравится · ❌ Отмена**.
2. **Нравится** → бот создаёт картинку под этот рецепт и публикует пост в канал.
3. **Не нравится** → можно написать, что изменить (одним сообщением), или нажать
   **⏭ Пропустить** — бот пересоздаёт превью (с учётом правок или просто другой вариант).
4. **Отмена** → черновик отменяется, бот возвращает к списку команд.

> Логи бота — `logs\bot.log`. Запускать можно только один экземпляр (Telegram вернёт
> ошибку 409, если слушателей несколько). `bot.bat` — разовый запуск без авто-перезапуска
> (для отладки); для постоянной работы используйте `launcher.bat`.

## Опрос управляет рецептами

Опрос (`/poll`) спрашивает аудиторию про приёмы пищи: **Завтрак / Обед / Ужин**. Пока бот
запущен (`launcher.py`), он считает голоса в реальном времени и пишет их в
`state/poll_pref.json`. При генерации рецепт учитывает результаты:

- **В совокупности:** распределение голосов передаётся модели — рецепты в целом смещаются
  под предпочтения аудитории.
- **Лидер → слот по времени:** победивший приём пищи получает «featured»-рецепт в свой слот.
  Завтрак → 09:00, Обед → 14:00, Ужин → 19:00. Время — **ближайшее подходящее**: если слот
  сегодня ещё впереди — сегодня, иначе — завтра (за счёт ежедневного расписания 09/14/19).

Текущие голоса посмотреть командой боту **`/pollresults`**.

> Чтобы учёт голосов работал, бот должен быть запущен (`launcher.py start`) — именно он
> слушает голоса. Опрос — анонимный, считается агрегатно (без привязки к людям).

## Запуск по расписанию (Планировщик Windows)

Запускается **один раз** — создаёт две задачи: `RecipesBot` (рецепты, 09:00 / 14:00 / 19:00
ежедневно) и `RecipesPoll` (опрос-вовлечение, воскресенье 12:00):

```powershell
powershell -ExecutionPolicy Bypass -File "C:\ИИ тг канал\setup_schedule.ps1"
```

«Умные» настройки: пропущенный запуск (если ПК был выключен) выполнится при включении,
ПК будит из сна, работает и от батареи, повтор при сбое.

- **Поменять расписание / число постов:** в `setup_schedule.ps1` измените `$Times` (например,
  5/день: `@("08:30","12:30","16:00","19:00","21:30")`) или `$PollDay`/`$PollTime`, запустите
  скрипт ещё раз. Рекомендация: старт 3 поста/день, потолок 5.
- **Проверить:** `Get-ScheduledTask RecipesBot, RecipesPoll`
- **Запустить сейчас:** `Start-ScheduledTask -TaskName RecipesBot` (или `RecipesPoll`)
- **Удалить:** `Unregister-ScheduledTask -TaskName RecipesBot,RecipesPoll -Confirm:$false`

> Задачи работают, когда вы вошли в систему. Если ПК часто выключен — перенесите запуск
> на дешёвый VPS + cron (тем же скриптом).

## Рост и вовлечение

Бот сам работает на распространение — настройки в `config.yaml`:
- **Кнопки** «Подписаться»/«Поделиться» под каждым постом (`buttons`).
- **Вотермарка** @канала на фото (`watermark`) — виден источник при репостах.
- **CTA** в тексте поста (`cta_text`).
- **Опрос** раз в неделю (`polls`, режим `--poll`, задача `RecipesPoll`).

Маркетинговый чек-лист (что делаете вы: упаковка канала, взаимопиар, посевы, метрики) —
в **[GROWTH.md](GROWTH.md)**.

## Настройка контента

- `config.yaml` → `rubric_by_hour` — рубрики по времени суток (утром завтраки, вечером
  ужины/десерты). Скрипт выбирает тему по текущему часу. `temperature` — разнообразие.
- `config.yaml` → `text_api` — провайдер текста (по умолчанию 1bitai, эндпоинт и модель
  настраиваются там же).
- `prompts/system_recipe.md` — тон, правила безопасности, шаблон фуд-фото, формат JSON.

## Монетизация

При ~1000+ подписчиков с нормальным ER — биржи **Telega.in** и **TGStat**,
форматы 1/24 и 2/24. Ориентир CPM для кулинарии — 5–6к₽.
